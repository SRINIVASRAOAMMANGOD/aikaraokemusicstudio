KEY FILES QUICK REFERENCE
==========================================

app.py — Routes map
  PAGE ROUTES
    GET  /                              → home.html (upload page)
    GET  /mixer/<project_id>            → mixer.html
    GET  /karaoke/<project_id>          → karaoke.html
    GET  /compare/<project_id>          → compare.html
    GET  /projects                      → projects.html
    GET  /settings                      → settings.html

  API ROUTES
    POST /upload                        → save file → run Demucs → create project → return redirect URL
    POST /upload-url                    → download from YouTube/URL → run Demucs → return redirect URL
                                          body: {url, model}  response: {success, redirect_url}
    GET  /api/stems/<project_id>        → return {stem_name: file_url} dict for mixer/karaoke
    GET  /api/project-requirements/<id> → {has_vocals, has_recording, vocal_url, recording_url}
    GET  /api/projects                  → list all projects with stems dict injected (sorted newest-first)
    GET  /api/projects/<id>             → get single project metadata
    DEL  /api/projects/<id>             → delete project folder entirely
    POST /api/projects/<id>/save-mix    → persist mixer volumes/mute/solo/tempo/pitch to metadata
    POST /api/save-recording/<id>       → save microphone recording blob, set has_recording=true
    POST /api/analyze-performance/<id>  → real librosa+mir_eval scoring; returns 6 metrics + overall
    GET  /api/settings                  → return merged settings (defaults + settings.json)
    POST /api/settings                  → persist settings to settings.json
    GET  /api/storage-info              → disk usage stats for projects/ folder
    POST /api/clear-cache               → wipe uploads/ and separated/ folders
    GET  /files/<path>                  → serve any project file securely (stems, recordings)

config.py — Environment configs
  DevelopmentConfig  DEBUG=True, no HTTPS, local SQLite
  ProductionConfig   DEBUG=False, requires SECRET_KEY env var
  TestingConfig      Separate test DB, CSRF disabled

services/demucs_service.py
  separate_audio(file_path, model, output_folder) → stems_path
  Module-level model cache prevents re-loading weights on every request
  Supports: htdemucs, htdemucs_ft, htdemucs_6s, mdx_extra, mdx_extra_q

services/url_service.py
  download_from_url(url, output_folder) → (file_path, title)  ← tuple, NOT bare string
  download_from_youtube()  uses fixed 'yt_audio' base name in outtmpl; captures real
    output path via postprocessor_hooks + progress_hooks; 3-strategy fallback resolution
  download_direct_audio()  direct HTTP download via requests
  is_youtube_url(), get_video_info()  helpers

services/scoring_service.py
  analyze_vocal_accuracy(vocal_path, recording_path) → dict
  Metrics (0-100): pitch_accuracy (pyin+mir_eval RPA), timing_accuracy (onset F-measure),
    tone_quality (MFCC cosine sim), expression (RMS Pearson), consistency (chroma accuracy),
    breath_control (pause-ratio similarity)
  Falls back to _fallback_analysis() if librosa/mir_eval not installed

settings.json (runtime, auto-created)
  Keys: audio_output, audio_input, sample_rate, buffer_size, normalize_audio,
        default_model, processing_quality, cpu_threads, use_gpu, auto_cleanup,
        theme, accent_color, waveform_colors, show_animations, compact_mode,
        max_storage, auto_delete_old, export_format, enable_debug, api_endpoint
